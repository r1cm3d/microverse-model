## Project Overview

Microverse Model is a Small Language Model (SLM) being built in Rust, trained on Rick and Morty episode transcripts (Seasons 1–8). The ML model itself (`src/`) is currently a placeholder — the only complete component is the data collection pipeline (`scripts/scraper.rs`).

## Commands

```bash
make build          # cargo build (dev)
make release        # cargo build --release
make test           # cargo test
make clippy         # cargo clippy -- -D warnings
make fmt            # cargo fmt
make run            # cargo run (main binary)
make run-example    # cargo run --example basic_usage
make scraper        # cargo run --bin scraper (re-scrapes fandom wiki)
make doc            # cargo doc --no-deps --open
```

To run a single test:
```bash
cargo test <test_name>              # e.g. cargo test test_parse_dialogue_line_simple
cargo test --bin scraper            # run all tests in scripts/scraper.rs
```

## Architecture

The project has two separate binaries defined in `Cargo.toml`:

- **`src/main.rs`** — future inference/training CLI (placeholder)
- **`scripts/scraper.rs`** — standalone data collection binary, declared via `[[bin]]` in `Cargo.toml`

**`src/lib.rs`** — future core model library (placeholder)

### Scraper binary (`scripts/scraper.rs`)

The only substantive code. It fetches Rick and Morty transcripts from `rickandmorty.fandom.com` and outputs structured data to `datasets/`. Key functions:

- `get_episode_list(season)` — fetches episode list HTML, delegates to `parse_episode_list_html`
- `scrape_episode(url, ...)` — fetches episode transcript HTML, delegates to `parse_episode_content_html`
- `parse_dialogue_line(text)` — splits `"Character: line"` format; rejects names ≥ 50 chars
- `write_dialogues_to_csv` — writes CSV; replaces `"` with `'` for escaping

The `DialogueLine` struct is the central data model: `{index, season, episode_no, episode, character, line}`.

Output files:
- `datasets/rick_morty_all_transcripts.csv` — flat CSV (committed)
- `datasets/rick_morty_all_transcripts.json` — rich JSON (generated by scraper, not committed)

### ML framework

Not yet selected. Candidates are `candle` (Hugging Face) and `tch-rs` (PyTorch bindings). The next major development phase is implementing tokenization and training loop in `src/`.
